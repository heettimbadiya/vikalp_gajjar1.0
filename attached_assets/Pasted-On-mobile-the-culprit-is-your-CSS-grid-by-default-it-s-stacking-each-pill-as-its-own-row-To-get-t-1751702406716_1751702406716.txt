On mobile the culprit is your CSS grid—by default it’s stacking each “pill” as its own row. To get the horizontal-scroll “pill” look you want, switch to a flex container on small screens:

// components/SolutionsHero.tsx
export default function SolutionsHero({ solutions }) {
  return (
    <section className="solutions-hero py-8 bg-gradient-to-r from-blue-700 to-blue-500 text-white">
      <div className="container mx-auto px-4">
        <h2 className="text-2xl sm:text-3xl font-bold mb-4 text-center">
          Explore Our Industry Solutions
        </h2>
-       <div className="solutions-grid overflow-x-auto snap-x snap-mandatory grid grid-cols-1 sm:grid-cols-2 lg:grid-cols-4 gap-4 px-2 scrollbar-hide">
+       <div
+         className="
+           solutions-grid
+           flex flex-nowrap gap-4   /* ← use flexbox on all screens */
+           sm:grid sm:grid-cols-2 lg:grid-cols-4  /* ← re-enable grid at ≥sm */
+           overflow-x-auto snap-x snap-mandatory px-2 scrollbar-hide
+         "
+       >
          {solutions.map((sol) => (
            <Link key={sol.slug} href={`/solutions/${sol.slug}`}>
              <a
                className="snap-center bg-white/20 hover:bg-white/30 text-white text-center py-3 rounded-full flex-1 whitespace-nowrap block focus:ring-2 focus:ring-white transition-all duration-300"
                aria-label={`Go to ${sol.name} solutions`}
              >
                {sol.name}
              </a>
            </Link>
          ))}
        </div>
      </div>
    </section>
  );
}

Why this works:
	•	flex flex-nowrap gap-4 on mobile ensures all pills sit in one row and can scroll horizontally.
	•	At sm: and up, you fall back to your grid so desktop remains unchanged.
